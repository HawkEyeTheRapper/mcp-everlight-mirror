## 2. Deploying on GitHub Pages

### Option A – Use the built HTML (no build step)
1. **Rebuild locally (optional)** – In a local clone, install the Sphinx dependencies:
   ```bash
   pip install sphinx myst-parser sphinx_rtd_theme
   cd docs
   make html       # or `sphinx-build -M dirhtml source build`
   ```
   Commit the contents of `docs/build`.
2. **Enable GitHub Pages** – In your repository, go to **Settings → Pages**.  
3. **Select the source** – Choose `main` as the branch and set the folder to `/docs/build` (or `/docs/build/html` if you used the default `html` builder).  GitHub Pages looks for an `index.html`, `index.md` or `README.md` in the selected folder【829239888632483†L156-L164】.
4. **Save and deploy** – After saving, GitHub’s GitHub Actions runner will publish the site.  It may take a few minutes; your site will be at `https://omniversal-media-llc.github.io/SphinxGuardian/`.  The built Sphinx site provides a sidebar and links to all your codex entries.

### Option B – Host the Markdown directly with Jekyll
If you don’t want to build with Sphinx and just want GitHub Pages to convert Markdown to HTML:
1. Create a `_config.yml` in the repository root with the following (from the “Easy Markdown to GitHub Pages” guide):
   ```yaml
   plugins:
     - jekyll-relative-links
   relative_links:
     enabled: true
     collections: true
   include:
     - README.md
     - LICENSE.md
     - metadata.json
     - Codex
   ```
2. Commit `_config.yml`, then enable GitHub Pages by selecting the `main` branch and root folder in **Settings → Pages**.
3. GitHub Pages will convert each `.md` file to `.html` automatically.  Your `README.md` or an `index.md` becomes the home page【829239888632483†L156-L164】.  Each codex file will be accessible by replacing `.md` with `.html`【163399832394846†L79-L87】.  Use relative links in your Markdown so links work both on GitHub and on your site.

## 3. Deploying on Cloudflare Pages

Cloudflare Pages can either run the Sphinx build or serve the built files directly.  When you create a Pages project:

1. **Connect to Git** – Log in to the Cloudflare dashboard, go to **Workers & Pages → Pages → Create a project**, and choose **Connect to Git**.  Select the `SphinxGuardian` repository.
2. **Configure builds and deployments** – In the setup screen, specify:
   - **Production branch:** `main`
   - **Build command:**  
     - To run Sphinx during deployment, set it to `make html` (the Cloudflare guide for Sphinx uses `make html`【130108633379752†L516-L534】).  
       You may need a `requirements.txt` with `sphinx`, `myst-parser` and `sphinx_rtd_theme`.  In **Environment variables**, add `PYTHON_VERSION` (for example, `3.7`)【130108633379752†L531-L542】.
     - If you prefer not to build on Cloudflare, set the build command to `exit 0` and commit the `docs/build` folder.  Cloudflare will then just deploy the static files【741625570361059†L246-L279】.
   - **Build output directory:** `docs/build/html` when using `make html` (or `docs/build` if you’re uploading the pre‑built site).  Cloudflare Pages needs an `index.html` at the top level of this directory; otherwise you’ll see a 404【741625570361059†L236-L279】.
3. **Deploy** – Save the configuration and start the deployment.  After the build finishes, Cloudflare will host your site at `<project-name>.pages.dev`, and it will be redeployed automatically whenever you push changes.

## 4. Common pitfalls
- **Missing index.html:** Both GitHub Pages and Cloudflare Pages require an `index.html` (or `index.md`/`README.md` on GitHub) at the root of the publishing directory【829239888632483†L156-L164】【741625570361059†L236-L279】.
- **Incorrect output directory:** For Sphinx, the default `make html` output is `build/html`; if you use the `dirhtml` builder (as in your Makefile), each page gets its own directory with an `index.html` and the top‑level output is still `build/html`.  Point your Pages configuration to that directory.
- **Uninstalled dependencies:** When building on Cloudflare, include a `requirements.txt` and set the `PYTHON_VERSION` environment variable so Cloudflare can install Sphinx and its extensions【130108633379752†L531-L542】.

By using the built Sphinx HTML or letting GitHub Pages/Jekyll convert your Markdown, you can publish **Sphinx Guardian** on either platform.
