When a `.zip` archive contains its own top‚Äëlevel folder (like `EverLightOS/`), `unzip` will faithfully recreate that folder. There isn‚Äôt a built‚Äëin flag to ‚Äústrip‚Äù just the first path component, but there are two common ways to achieve what you want:

1. **Extract normally, move, then remove the wrapper folder**:

   ```bash
   # extract the zip (creates EverLightOS/ in the current dir)
   unzip EverLightOS_scaffold.zip

   # move everything inside EverLightOS into the current directory
   mv EverLightOS/* .

   # remove the now-empty top-level directory
   rmdir EverLightOS
   ```

   This preserves the directory structure (`core/`, `styles/`, etc.) but removes the extra top‚Äëlevel wrapper.

2. **Use the `-j` (‚Äújunk paths‚Äù) flag to flatten everything** (not recommended if you need sub‚Äëfolders):

   ```bash
   unzip -j EverLightOS_scaffold.zip
   ```

   With `-j`, all files are dumped into the current directory and any subdirectory structure is discarded. That‚Äôs usually only useful when there are no nested folders you want to keep.

If you‚Äôre on a system with BSD tar (Git Bash typically includes it), there‚Äôs also a one‚Äëliner that strips just the first directory layer:
